Maybe Redux confuses you. Even if you’ve been using it for a bit. When I talk with Junior Developers, it consistently ranks as one of the most difficult to understand parts of working with ReactJS. It doesn’t have to be.

Now, let’s dive in.

What is Redux Really?

A predictable state container. But what does that mean?

Whether you’re using Angular, Vue or React you will have to manage the state/data for your components. In React you may be using props or state to share data between components. In more complex applications eventually managing local state and sharing data across components may get unwieldy.

In order to centralize your state logic you can leverage a library like Redux. This has the additional benefit of creating some separation between your presentational and state logic.

Why Does Redux Confuse So Many Developers?

TLDR; Redux has a lot of boilerplate and tutorials usually leverage a weird pattern.

Setting up Redux in a React app can be a chore. Then you head to a tutorial and you are overwhelmed by trying to keep track of concepts like actions, action types, actions creators, and reducers which of course all live in separate files. There’s a better way to structure your Redux logic which you’ll see in the tutorial at the end of this article.

The Pattern You Need to Know Before Redux

You don’t understand Redux because you don’t know the Observer pattern.

I was confused by Redux for years, even after using it for quite some time. It just seemed magical. Luckily a more senior developer shared his mental model with me and sent me an article about the Observer pattern. It clicked!

Check out the Observer pattern here and try to implement it on your own!

What About Framework [X]?

There are better alternatives to Redux, right?

Depends on who you ask. But, yes, there are quite a few other libraries out there which are gaining traction. Recoil, Zustand, and React context and hooks CAN be used in place of Redux in a React app.

So why are we still learning Redux?

Redux is the undisputed King of the state management world and is the mainstream choice for enterprise applications. You will almost certainly encounter it and the patterns it leverages are similar in other libraries you will use.
