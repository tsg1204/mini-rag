Default Parameters

If you have a function which depends on an argument being present and being of a certain type then you should consider using default parameters in your functionâ€™s signature (where you add the arguments).

If either one of these conditions is not met, we will fail spectacularly.

Using An Object as anÂ Argument

The order of your arguments in a function matter! Having more than a couple arguments means that developers need to be careful how they are calling your function. Imagine having a function with 6 or 8 args! Itâ€™s too easy to slip up and believe me, someone will.

Before

  // Logic to create user
}
createUser('Alice', 30, 'alice@example.com');

The danger with functions like these are that the order of the arguments matters and if any of them are incorrect, we risk blowing up our program.

After

  // Logic to create user
}
createUser({ name: 'Alice', age: 30, email: 'alice@example.com' });

Boom. Using an object, we make sure that our arguments are explicitly set so the order no longer matters. For functions with more than 3 arguments, I feel like using an object is a must.
